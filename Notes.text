That’s a **great assignment**—and it’s exactly the kind of thing companies use to test if you can build *real AI products*, not just call APIs.

But “dual model GenAI assistant” isn’t just two APIs glued together. If you do that, it’ll feel pointless. The real goal is:

👉 **Use two models intelligently for different strengths**

---

# 🧠 What “Dual Model GenAI Assistant” actually means

Use:

* One model for **research (factual, structured, retrieval-heavy)**
* One model for **creative writing (style, storytelling, tone)**

---

## 🔥 Recommended Setup

### Model 1 (Research)

* OpenAI (GPT models)
  **Why:**
* Better structured outputs
* Function calling
* Good for factual answers

---

### Model 2 (Creative Writing)

* Anthropic (Claude)
  **Why:**
* Better long-form writing
* More natural tone
* Handles large context well

---

# 🧩 Core Features Your Assistant Should Have

## 1. 🔍 Research Mode

User asks:

> “Explain microservices architecture”

System:

* Uses OpenAI
* Returns:

  * Summary
  * Key points
  * Sources (optional)

---

## 2. ✍️ Creative Mode

User asks:

> “Write a blog on microservices in a storytelling style”

System:

* Uses Claude
* Returns:

  * Engaging blog
  * Natural tone

## 3. 🔄 Smart Routing (IMPORTANT)

Instead of user choosing manually, you can:

### Option A (basic):

```js
if (query.includes("write") || query.includes("story")) {
  useClaude();
} else {
  useOpenAI();
}
```

### Option B (better):

Use LLM to classify intent:

* research vs creative
